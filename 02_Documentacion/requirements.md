# Requerimientos del Proyecto: Cajero Automático

## 1. Nombre del proyecto

Sistema de Cajero Automático

## 2. Objetivo general

Desarrollar una aplicación de escritorio que simule el funcionamiento básico de un cajero automático, permitiendo al usuario iniciar sesión, consultar saldo, realizar depósitos, retiros y visualizar movimientos de su cuenta.

## 3. Tecnologías utilizadas

* Java
* Java Swing
* Maven
* Programación Orientada a Objetos
* SQL Server
* JDBC
* IntelliJ IDEA
* Git y GitHub

## 4. Requerimientos funcionales

### RF01 - Inicio de sesión

El sistema debe permitir que un usuario ingrese con su número de cuenta o usuario y su PIN correspondiente.

### RF02 - Validación de usuario

El sistema debe validar que los datos ingresados sean correctos antes de permitir el acceso al menú principal.

### RF03 - Menú principal

El sistema debe mostrar un menú con las opciones principales del cajero automático.

### RF04 - Consulta de saldo

El sistema debe permitir consultar el saldo disponible de la cuenta del usuario.

### RF05 - Depósito de dinero

El sistema debe permitir realizar depósitos y actualizar el saldo de la cuenta.

### RF06 - Retiro de dinero

El sistema debe permitir realizar retiros siempre que el usuario tenga saldo suficiente.

### RF07 - Registro de movimientos

El sistema debe registrar las operaciones realizadas por el usuario, como depósitos y retiros.

### RF08 - Cierre de sesión

El sistema debe permitir cerrar sesión y regresar a la pantalla de inicio.

## 5. Requerimientos no funcionales

### RNF01 - Interfaz gráfica

El sistema debe contar con una interfaz gráfica sencilla y clara utilizando Java Swing.

### RNF02 - Organización del código

El código fuente debe estar organizado en paquetes y clases según la Programación Orientada a Objetos.

### RNF03 - Base de datos

El sistema debe utilizar SQL Server para almacenar usuarios, cuentas y movimientos.

### RNF04 - Seguridad básica

El PIN del usuario no debe mostrarse en pantalla al momento de iniciar sesión.

### RNF05 - Control de errores

El sistema debe mostrar mensajes de error cuando los datos sean incorrectos o cuando una operación no pueda realizarse.

## 6. Actores del sistema

* Usuario del cajero automático
* Sistema

## 7. Casos de uso principales

1. Iniciar sesión.
2. Consultar saldo.
3. Depositar dinero.
4. Retirar dinero.
5. Ver movimientos.
6. Cerrar sesión.
